# Overview

Starting a new project can be one of the best moments in the project itself. The codebase is clean, everything is simple and makes sense. Usually when getting strated developers have a preference in a styling system whether it be **Styled Components**, **SCSS** or in this case **TailwindCSS**.

Alongside using TailwindCSS I also start off most projects with at least something from **ShadcnUI**, at minimum the beautiful toast component.

Installing Shadcn changes your **globals.css** as well as your **tailwind.config.ts** files, and as the project grows I find that things can get a little bit crowded. This led me to scouring the internet for solutions when I finally stumbled across something I fell in love with and implement to this day, _creating a tailwindcss plugin with all my shadcn styles_.

Below are the changes I make to my app structure so that I can implement a simple tailwind plugin and all my shadcn stuff is in one place. You can check out the files below.

```jsx
// file - globals.css

@tailwind base;
@tailwind components;
@tailwind utilities;

@layer utilities {
  /* Hide scrollbar for Chrome, Safari and Opera */
  .scrollbar-hide::-webkit-scrollbar {
    display: none;
  }

  /* Hide scrollbar for IE, Edge and Firefox */
  .scrollbar-hide {
    -ms-overflow-style: none; /* IE and Edge */
    scrollbar-width: none; /* Firefox */
  }
}

```

```jsx
// file - shadcnPreset.ts

// types
import type { Config } from "tailwindcss";

// plugins
import animatePlugin from "tailwindcss-animate";
import { shadcnPlugin } from "./shadcnPlugin";

export const shadcnPreset = {
  darkMode: ["class"],
  content: [],
  plugins: [animatePlugin, shadcnPlugin],
} satisfies Config;
```

```jsx
// file - tailwind.config.ts

// types
import type { Config } from "tailwindcss";

// utils
import { shadcnPreset } from "./src/lib/theme/shadcnPreset";

const config = {
  presets: [shadcnPreset],
  content: [
    "./pages/**/*.{ts,tsx}",
    "./components/**/*.{ts,tsx}",
    "./app/**/*.{ts,tsx}",
    "./src/**/*.{ts,tsx}",
  ],
  theme: {
    extend: {
      screens: {
        xs: "400px",
      },
    },
  },
  plugins: [
    require("@tailwindcss/container-queries"),
  ],
} satisfies Config;

export default config;
```
