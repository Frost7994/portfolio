# Overview

As you may have guessed if you've looked through this portfolio, I love to implement framer motion wherever I can as long as it's appropriate. However, one of the downfalls I found working with it was there was no easy way to animate a component on page load and thats it. Something like the opposite of initial={false} on AnimatePresence, I would love a prop like animateOnce={true}.

Since there is no solution like this as of yet, I created a custom hook to fill the void.

The idea is simple, to track a timestamp using local storage and when it expires, reanimate. The hook exposes a value as a boolean which you can use in your code to animate like so:

```jsx
// utils
import { useIntro } from "@/lib/hooks/useIntro";
import { motion } from "framer-motion";

const Component = () => {
  const showAnimation = useIntro();

  return (
    <motion.div
      initial={{
        opacity: showAnimation ? 0 : 1,
      }}
      animate={{
        opacity: 1,
      }}
      exit={{
        opacity: 0,
      }}
      className="h-12 w-12 bg-primary rounded-md"
    />
  );
};
```

The hook is as follows:

```jsx
"use client";

// state
import { useEffect } from "react";

export const useIntro = () => {
  const storage = window.localStorage;
  const currentTimestamp = Date.now();
  const timestamp = storage.getItem("tommybyrne-timestamp") || "1000";

  const timeLimit = 5 * 60 * 1000; // 5 mins
  const hasTimePassed = currentTimestamp - parseInt(timestamp) > timeLimit;

  useEffect(() => {
    hasTimePassed
      ? storage.setItem("tommybyrne-timestamp", currentTimestamp.toString())
      : storage.setItem("tommybyrne-timestamp", timestamp.toString());
  }, []);

  return hasTimePassed;
};
```
